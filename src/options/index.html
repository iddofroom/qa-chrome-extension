QA Assistant — הגדרות

API Secret

נשלח כ-Authorization: Bearer <secret> לכל ה-endpoints. נשמר רק ב-chrome.storage.local שלך.

GitHub PAT (אופציונלי — לייבוא repos)

טוקן read-only כדי לייבא רשימת repos שלך עם שדה homepage מלא. fine-grained PAT עם הרשאת Repository → Metadata → Read מספיק. הטוקן נשלח רק ל-api.github.com ונשמר רק ב-chrome.storage.local. אם תשאיר ריק, התכונה הזאת לא פעילה.

פרוייקטים

לכל פרוייקט: תווית (לזיהוי), Origin (Chrome match pattern של האתר שאתה בודק — למשל https://example.com/*; אם תכניס רק https://example.com או https://example.com/ זה יושלם אוטומטית ל-/*), ו-Endpoint URL (איפה לשלוח את הבקשה — חייב להיות https://, חוץ מ-localhost). בעת השמירה הדפדפן יבקש הרשאה לכל origin חדש; הרשאות שלא בשימוש יוסרו.

ייבוא / ייצוא JSON

פורמט: [{"label": "...", "origin": "...", "endpoint": "..."}]. ייצוא ממלא את התיבה למטה מהטבלה הנוכחית. ייבוא מוסיף לטבלה את כל הרשומות התקינות שב-JSON (לא מחליף — רק מצרף; שורות כפולות תצטרך למחוק ידנית). ה-API secret לא נכלל בייצוא, מסיבות אבטחה.

"משוך מ-GitHub" קורא ל-/user/repos עם ה-PAT למעלה, מסנן ל-repos עם homepage ממולא, ומציג אותם בתיבה. ה-endpoint מוגדר אוטומטית כ-{homepage}/api/qa-assistant — אם ה-path אצלך שונה, ערוך לפני "ייבוא מהתיבה".
